# 入力の組み合わせ別: メロディ+歌詞タイミング情報の取得手法

NEUTRINO歌唱合成に必要な情報と、入力（歌唱音源 / XF MIDI / 非XFメロディMIDI /
元歌詞表層 / 元歌詞読み）の組み合わせごとの取得手法の整理。
実測値はすべて夜に駆ける（275秒、歌詞51行）でのXF正解評価による。

## 必要な情報（ゴール）

NEUTRINOに歌わせる（=MusicXMLを作る）には以下が揃えばよい:

| 記号 | 情報 | 備考 |
|---|---|---|
| P | 音符列の音高 | 半音単位。移調は一律オフセットで可 |
| T | 音符列のタイミング | 開始・長さ。テンポは固定BPM1本+休符埋めで足りる(厳密なテンポ復元は不要) |
| K | モーラ列（歌唱カナ） | 発音基準のカナ |
| A | モーラ↔音符の対応 | 1モーラ1音符が基本。同音連打の連結・メリスマ(1モーラ複数音符=ー継続)あり |

さらに動画まで作るなら: 伴奏音源（mix用）、元歌詞表層（字幕用）。

## 各入力が持っている情報

| 入力 | P | T | K | A | 備考 |
|---|---|---|---|---|---|
| XF MIDI | ◎ | ◎ | ◎ | ◎ | 全部入りの正解。これがあれば推定は不要 |
| 非XFメロディMIDI | ◎ | ○ | × | × | Pは正確。Tは楽譜時間軸(音源とはオフセット+テンポ差あり)。メロディchの特定が必要 |
| 歌唱音源 | △ | ○ | △ | △ | f0でP(一致81%)、CTCでモーラ開始(MAE24ms)、Whisperで歌詞、ただし全部「推定」 |
| 元歌詞表層 | × | × | ○ | × | G2P(読み推定)を介してK。読み誤りが残る(明日=アシタ/アス等は原理的に不定) |
| 元歌詞読み | × | × | ◎ | × | Kの正解。読み推定と音響判定が不要になる |

ポイント: **音源は「正解」でなく「検証・アンカー」として使うのが正しい**。
音源から推定した値(f0・CTC時刻)を基準に楽譜を検品すると逆方向の修正をする
(実測で経験済み)。構造が正しいのは楽譜(MIDI)側。

## 組み合わせ別の手法

「少なくとも音源かXF MIDIのどちらかがある」前提で、主要な組み合わせを品質順に並べる。

### 1. XF MIDI（+任意で音源・表層）→ 最良・確定的

- P/T/K/A すべてXFから読むだけ（`analyze`）。推定ゼロ
- 表層があれば字幕用に行アライメント（difflib）
- 音源があれば: demucs分離した伴奏をmixに使える。ただしXFのタイムラインと
  音源のタイムラインはイントロ差等でずれるので、大域線形写像
  （オフセット+テンポスケール、`fit_global_map`と同じ格子探索）で合わせる
- 音源が無ければ伴奏はMIDIレンダリング（fluidsynth）。ずれの問題自体が消える

### 2. 音源 + 非XFメロディMIDI + 表層（or 読み）→ 実用ライン(現行 `analyze-audio --melody-midi`)

実測: **ピッチ98-99% / タイミングMAE 1-4ms / 被覆率98%**

1. demucsで vocals / no_vocals に分離
2. K: 表層→読み推定(soramimic-yomi)。読み候補が割れた行はCTC対数尤度
   (対数ベイズ因子、閾値2.0)で音響判定。**読みが入力にあればこの推定・判定は丸ごと不要**
3. モーラ開始時刻: CTC forced alignment（wav2vec2 rs35kh）
4. MIDI→音源の時間写像: 大域線形（オフセット+スケール）を格子探索。
   クロマDTWのような自由度の高い写像は粒度(約0.2秒)がモーラ間隔と同オーダーで
   対応を壊すため使わない（実測でピッチ一致50%まで劣化した）
5. A: CTCモーラ開始列 × 写像済み音符開始列の単調DP（時刻+音高コスト、
   多対一=同音連打の音符共有を許可）。メロディchは「被覆率+音高輪郭MAD」で自動選択
6. P/T: 対応がついたモーラは**MIDIの音高とnote-on/offをそのまま採用**
   （移調はf0との中央値差で自動補正）。CTC時刻は対応付けと連打分割にだけ使う。
   余り音符はメリスマ(ー)、余りモーラはf0フォールバック

制約: テンポ変化する曲は線形写像では追えない（区分線形化が将来課題）。
メロディが独立チャンネルに無いMIDI（skyline頼み）は品質が落ちる（実測: 被覆率53%）。

### 3. 音源 + 表層（or 読み）→ MIDIなしの推定経路

実測: **ピッチ完全一致84% / ±1半音88% / タイミングMAE 26ms**

- K: 2.と同じ（読み入力があれば確定）
- T: CTCモーラ開始 + 有声区間(pyin)による終端伸長。行タイミングは十分実用
- P: f0(pyin)のモーラ区間の**モード(最頻半音)**。中央値はビブラート・しゃくり・
  リリースのピッチdriftに引っ張られて隣の半音へ外れるが、モードは最も長く
  保持された半音を拾うため頑健（XF正解評価: 中央値81%→モード84%）
- A: 自明（1モーラ1音符に割り切る）

**試したが効かなかったもの（issue #4/#5、eval-audioで検証）**:
- キー推定+スケール吸着: 81%のまま。夜に駆けるは半音階的でスケール外音が
  「誤りでなく本物」のため、吸着すると正しい音を壊す（tie強制だと78%に悪化）
- 歌唱採譜(RMVPE+ROSVOT)でpseudo-MIDI: ROSVOTのノート分割は弱い音節を
  取りこぼし被覆57%。RMVPEのf0自体は綺麗だが、モード集約と組み合わせても
  実パイプラインでpyin+モード(84%)と同じ84%で、重い外部依存に見合わない
- いずれも決定打はモード集約(1行)で、±1半音の外しは推定の限界。99%には
  外部の独立した楽譜(2.)が要る

### 4. 音源のみ（歌詞なし）→ フォールバック

- K: Whisper（faster-whisper large-v3）の認識結果を歌詞代わりに使う
  （セグメント=行）。認識誤りは替え歌変換の入力誤りとして残る
- 以降は3.と同じ。品質は3.以下

### 5. メロディMIDI + 読み（音源なし）→ `analyze-midi`(実装済み)

生成メロディ(ChatMusician等)や打ち込みメロディから、音源もf0も使わず器を作る。
著作権フリーの全自動替え歌(生成メロディ+生成歌詞→空耳→生成画像→NEUTRINO)の起点。

- P/T: MIDIそのまま(=確定)。伴奏はメロディMIDIをそのままレンダリング
  (音源アンカー不要。sf2があれば伴奏付き、無ければボーカルのみ)
- A: 音響が無いので構造だけで対応。**1音符=1モーラ**に割り切る
  (和音混じりは skyline で旋律線化、休符 or 最大音符数でフレーズ=行に分割)
- K: ベース歌詞をカナ化して音符に流し込む(soramimic-yomi)。無ければ「ラ」で充填。
  以降の convert が空耳変換する
- 実測: 夜に駆けるのような品質評価は不要(楽譜がそのまま正解)。歌唱の自然さは
  生成メロディの質(音符長・フレーズ感・刻みの少なさ)に依存する

## 横断的な設計原則（実測からの教訓）

1. **構造が正しい入力を基準にする**: 楽譜(MIDI/XF) > 読み入力 > 音源からの推定。
   推定値(f0・CTC)で楽譜を検品しない
2. **音源の役割は3つ**: 伴奏素材 / タイミングのアンカー(モーラ開始・大域写像の推定) /
   曖昧性の音響判定(読み候補の尤度比較)
3. **楽譜への補正は低自由度に限る**: 移調(一律) + テンポスケール + オフセット
   (+フレーズ頭の休符)。自由度の高い写像(DTW)や音符単位の時刻ブレンドは
   ジッタ・取り違えの持ち込みになる
4. **正解が作れる組み合わせ(XF+音源)で評価ハーネスを先に作る**: `eval-audio`。
   試聴と間接指標だけのチューニングは逆方向に進み得る
